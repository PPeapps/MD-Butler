# Contributing

Vielen Dank für dein Interesse, zu diesem Projekt beizutragen!

## Bugs melden

Bitte nutze GitHub Issues und beschreibe: - Erwartetes Verhalten -
Tatsächliches Verhalten - Schritte zur Reproduktion - Obsidian-Version -
Plugin-Version

## Feature Requests

Neue Ideen sind willkommen. Bitte beschreibe den Anwendungsfall
möglichst konkret.

## Pull Requests

-   Erstelle einen Feature-Branch.
-   Halte Pull Requests möglichst klein.
-   Aktualisiere bei Bedarf die Dokumentation.
-   Achte darauf, dass der Code erfolgreich gebaut werden kann.

Vielen Dank für deinen Beitrag!
